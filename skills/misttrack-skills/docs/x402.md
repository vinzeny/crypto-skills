# x402 Payment Protocol

> An open-source payment protocol developed by Coinbase that revives the HTTP `402 Payment Required` status code to enable instant, automated payments using stablecoins (USDC).

## Overview

x402 is a modern payment protocol designed to address the challenges traditional payment systems face in the digital economy and AI era. It leverages blockchain technology and stablecoins (USDC) to achieve:

- Instant Settlement: Payments confirmed in seconds
- Minimal Fees: Transaction fees around 0.0001
- No Account Required: Users can pay without registration
- Machine-Native: Built for automated payment scenarios

---

## Key Advantages

### Comparison with Traditional Payments

| Feature | Traditional Payments | x402 |
|---------|---------------------|------|
| Transaction Fees | 2-3% | 0.0001 |
| Settlement Time | Days | 200ms - 2s |
| Account Required | Yes, registration needed | No account needed |
| Machine Payments | Not supported | Native support |
| Cross-border | Complex & expensive | Seamless & low-cost |

### Technical Features

- EIP-3009 Based: Uses authorization transfer mechanism, no pre-funding required
- Replay Protection: Each transaction includes unique nonce and expiration
- Multi-Chain Support: Base, Solana, Ethereum
- Standard Compatible: Reuses HTTP 402 status code for easy integration

## Architecture

### Three-Party Architecture

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   Client    │ ───▶ │   Server    │ ───▶ │ Facilitator │
│  (Payer)    │ ◀─── │  (Provider) │ ◀─── │ (Settlement)│
└─────────────┘      └─────────────┘      └─────────────┘
      │                                          │
      └──────────────────────────────────────────┘
                    Blockchain Confirmation
```

| Role | Responsibility | Example |
|------|----------------|---------|
| Client | Initiates request and payment | AI Agent, Developer App |
| Server | Provides paid resources | API Service, Content Platform |
| Facilitator | Verifies on-chain payment | Coinbase CDP, PayAI |

### Payment Instruction Format

When server returns `402 Payment Required`, the response body contains:

```json
{
  "type": "x402",
  "amount": "0.05",
  "currency": "USDC",
  "facilitator": "https://facilitator.cdp.coinbase.com",
  "recipient": "0xYourAddress...",
  "network": "base",
  "expires": "2026-03-18T14:00:00Z"
}
```

| Field | Description | Example |
|-------|-------------|---------|
| `type` | Protocol type identifier | `"x402"` |
| `amount` | Payment amount (string) | `"0.05"` |
| `currency` | Token type | `"USDC"` |
| `facilitator` | Settlement service URL | `"https://facilitator.cdp.coinbase.com"` |
| `recipient` | Recipient address | `"0x..."` |
| `network` | Blockchain network | `"base"`, `"base-sepolia"`, `"solana"` |
| `expires` | Expiration time (ISO 8601) | `"2026-03-18T14:00:00Z"` |

## Protocol Flow

### Complete Interaction Flow

```
Step 1: Client ──GET /api/resource──▶ Server

Step 2: Client ◀──402 + x402 instruction── Server

Step 3: Client signs payment authorization (EIP-3009)

Step 4: Client ──Request + X-Payment header──▶ Server

Step 5: Server ──Verification request──▶ Facilitator

Step 6: Facilitator ──On-chain confirmation──▶ Blockchain

Step 7: Client ◀──200 OK + Resource── Server
```

### HTTP Status Codes

| Status Code | Meaning | Description |
|:-----------:|---------|-------------|
| `402` | Payment Required | Payment needed, response contains payment instruction |
| `200` | OK | Payment successful, returns requested resource |
| `409` | Conflict | Payment already processed (replay protection) |

## Server Integration

### FastAPI Example

```python
from typing import Any

from fastapi import FastAPI

from x402.http import FacilitatorConfig, HTTPFacilitatorClient, PaymentOption
from x402.http.middleware.fastapi import PaymentMiddlewareASGI
from x402.http.types import RouteConfig
from x402.mechanisms.evm.exact import ExactEvmServerScheme
from x402.server import x402ResourceServer

app = FastAPI()

# Your receiving wallet address
pay_to = "0xYourAddress"

# Create facilitator client (testnet — change URL for production)
facilitator = HTTPFacilitatorClient(
    FacilitatorConfig(url="https://x402.org/facilitator")
)

# Register EVM scheme
server = x402ResourceServer(facilitator)
server.register("eip155:84532", ExactEvmServerScheme())  # Base Sepolia

# Define protected routes
routes: dict[str, RouteConfig] = {
    "GET /weather": RouteConfig(
        accepts=[
            PaymentOption(
                scheme="exact",
                pay_to=pay_to,
                price="$0.001",           # Dollar-prefixed string, required
                network="eip155:84532",   # CAIP-2 format
            ),
        ],
        mime_type="application/json",
        description="Get current weather data for any location",
    ),
}

app.add_middleware(PaymentMiddlewareASGI, routes=routes, server=server)


@app.get("/weather")
async def get_weather() -> dict[str, Any]:
    return {"report": {"weather": "sunny", "temperature": 70}}


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=4021)
```

### Manual EIP-3009 Flow (reference)

The client-side flow without a SDK:

```python
import requests

# Step 1: Send initial request
response = requests.get("https://api.example.com/resource")

if response.status_code == 402:
    # Step 2: Parse X-Payment-Required header (base64 JSON)
    payment_info = response.headers.get("X-Payment-Required")

    # Step 3: Sign EIP-3009 authorization off-chain
    # (see scripts/pay.py for the full EIP-712 signing implementation)
    payment_payload = sign_eip3009(payment_info)

    # Step 4: Retry with X-Payment header
    response = requests.get(
        "https://api.example.com/resource",
        headers={"X-Payment": payment_payload}
    )

print(response.json())
```

## FAQ

**Q: Is x402 a token?**

A: No. x402 is a payment protocol that uses existing stablecoins like USDC for settlement. It does not issue its own token.

**Q: How much does it cost to use x402?**

A: The main cost is blockchain gas fees. On Base network, a single transaction costs approximately 0.0001, significantly lower than traditional payment fees of 2-3%.

**Q: How is replay attack prevented?**

A: x402 uses the EIP-3009 authorization transfer standard, which includes:
- Unique nonce value per transaction
- Expiration timestamp
- On-chain verification ensuring each authorization can only be used once

**Q: Which wallets are supported?**

A: Any wallet supporting EIP-3009 signing (EVM chains) or Solana signatures can be used, including:
- MetaMask
- Coinbase Wallet
- Phantom (Solana)
- Various custodial wallet solutions

**Q: How to get test tokens on testnet?**

A:
- Base Sepolia: Use [Coinbase Faucet](https://portal.cdp.coinbase.com/products/faucet) for test USDC
- Solana Devnet: Use official faucet

## 概述

x402 是一种现代化的支付协议，旨在解决传统支付系统在数字经济和 AI 时代面临的挑战。它利用区块链技术和稳定币（USDC）实现：

- 即时结算：支付在秒级完成确认
- 极低费用：交易费用约为 0.0001
- 无需账户：用户无需注册即可支付
- 机器原生：专为自动化支付场景设计

## 核心优势

### 与传统支付对比

| 特性 | 传统支付 | x402 |
|------|---------|------|
| 交易费用 | 2-3% | 0.0001 |
| 结算时间 | 数天 | 200ms - 2s |
| 账户要求 | 需注册开户 | 无需账户 |
| 机器支付 | 不支持 | 原生支持 |
| 跨境支付 | 复杂且昂贵 | 无缝且低成本 |

- 基于 EIP-3009：使用授权转账机制，无需预存资金

- 多链支持：Base、Solana、Ethereum
- 标准兼容：复用 HTTP 402 状态码，易于集成

## 协议流程

### 完整交互流程

```
步骤 1: Client ──GET /api/resource──▶ Server

步骤 2: Client ◀──402 + x402 指令── Server

步骤 3: Client 签名支付授权 (EIP-3009)

步骤 4: Client ──请求 + X-Payment 头──▶ Server

步骤 5: Server ──验证请求──▶ Facilitator

步骤 6: Facilitator ──链上确认──▶ Blockchain

步骤 7: Client ◀──200 OK + 资源── Server
```

### HTTP 状态码

| 状态码 | 含义 | 说明 |
|:------:|------|------|
| `402` | Payment Required | 需要支付，响应体包含支付指令 |
| `200` | OK | 支付成功，返回请求资源 |
| `409` | Conflict | 支付已处理（防重放保护） |

    # 第二步：解析 X-Payment-Required 响应头（base64 JSON）
    payment_info = response.headers.get("X-Payment-Required")

    # 第三步：离链签名 EIP-3009 授权
    # （完整的 EIP-712 签名实现见 scripts/pay.py）
    payment_payload = sign_eip3009(payment_info)

    # 第四步：携带 X-Payment 头重试请求
    response = requests.get(
        "https://api.example.com/resource",
        headers={"X-Payment": payment_payload}
    )

print(response.json())

## 常见问题

**Q: x402 是一种代币吗？**

A: 不是。x402 是一种支付协议，使用 USDC 等现有稳定币进行结算。它本身不发行代币。

**Q: 使用 x402 需要支付多少费用？**

A: 主要费用是区块链 Gas 费。在 Base 网络上，单笔交易费用约为 0.0001，远低于传统支付的 2-3% 手续费。

**Q: 如何防止重放攻击？**

A: x402 使用 EIP-3009 授权转账标准，每笔交易包含：
- 唯一 nonce 值
- 过期时间戳
- 链上验证确保每笔授权只能使用一次

**Q: 支持哪些钱包？**

A: 任何支持 EIP-3009 签名（以太坊系）或 Solana 签名的钱包都可以使用。包括：
- MetaMask
- Coinbase Wallet
- Phantom (Solana)
- 以及各种托管钱包解决方案

**Q: 测试网如何获取测试代币？**

A:
- Base Sepolia: 使用 [Coinbase Faucet](https://portal.cdp.coinbase.com/products/faucet) 获取测试 USDC
- Solana Devnet: 使用官方水龙头
