# Correctness Rubric for v4-sdk-integration

Evaluate whether the generated output correctly implements Uniswap v4 SDK integration.

## Required Elements (Must Include)

Score based on how many of these are correctly implemented:

1. **Correct SDK Imports**
   - Imports from `@uniswap/v4-sdk` (V4Planner, Actions, Pool, etc.)
   - Imports from `@uniswap/sdk-core` for token/currency types
   - Imports from `@uniswap/universal-router-sdk` when constructing swap calldata

2. **V4Planner and RoutePlanner Usage for Swap Execution**
   - Uses V4Planner to add swap actions (not direct PoolManager calls)
   - Encodes plan and passes to RoutePlanner with CommandType.V4_SWAP
   - Calls Universal Router execute() with the encoded commands and inputs

3. **Quoter callStatic Usage (Not Onchain)**
   - Uses Quoter contract with callStatic (not a live onchain transaction)
   - Calls appropriate quote method (quoteExactInputSingle, quoteExactInput, etc.)
   - Does NOT attempt to call Quoter as a state-mutating transaction

4. **Permit2 Two-Step Approval for ERC20**
   - Approves the token to the Permit2 contract address (not directly to router)
   - Grants Permit2 allowance to the Universal Router as the second step
   - Correctly identifies native ETH as exempt from the Permit2 flow

5. **Pool.getPoolId() for Pool Identifier Computation**
   - Uses `Pool.getPoolId()` to compute pool identifiers
   - Does NOT construct raw pool IDs by hand without using the SDK helper
   - Pool key is correctly constructed with currency0, currency1, fee, tickSpacing, hooks

## Scoring Guide

- 5/5 elements correct: 1.0
- 4/5 elements correct: 0.8
- 3/5 elements correct: 0.6
- 2/5 elements correct: 0.4
- 1/5 elements correct: 0.2
- 0/5 elements correct: 0.0

## Instructions

Count how many of the required elements are correctly implemented and provide the corresponding score.
